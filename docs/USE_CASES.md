# Use Cases

## Edge Analytics

QRD is designed for devices that generate structured telemetry locally and need to store or forward compressed, schema-aware data.

## Browser Telemetry

Browser clients can produce and consume QRD files with WASM bindings, enabling local analytics and offline diagnostics.

## Offline Inference

QRD supports local feature storage for model input preparation and cached inference data.

## Cross-Language Exchange

QRD is intended for environments where the same data file is produced and consumed by multiple runtimes without reimplementation drift.

## Compliance and Audit Logging

Self-describing schema and digest metadata make QRD useful for audit-ready telemetry and deterministic log archive storage.
